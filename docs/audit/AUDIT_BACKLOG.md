- **Дата выполнения:** 2026-06-01
- **Проблема:** Нет данных о DAU, MAU, CTR, delivery_success, FLOOD_WAIT. Бот работает вслепую.
- **Критерий приёмки:**

  - [x] Таблица `delivery_errors` с типами ошибок: FLOOD_WAIT, NetworkError, TelegramAPIError, UnknownError
  - [x] Таблица `user_sessions` для расчёта DAU/MAU
  - [x] Команда `/analytics` для админа с выводом: DAU (1д), MAU (30д), delivery rate (24ч/7д), ошибки, топ-10 источников
  - [x] Логирование всех `TelegramAPIError` с кодом ошибки (особенно 429 FLOOD_WAIT)
- **Файлы:** `storage/analytics.py` (330 строк), `tests/test_analytics.py` (9 тестов)
- **Теги:** `analytics`, `metrics`, `data`

### ✅ P0-002: Асинхронный перевод — **ВЫПОЛНЕНО**
- **Дата выполнения:** 2026-06-01
- **Проблема:** `translator.py` использует синхронный `httpx.Client()`, блокируя event loop. При 15 источниках и переводе summary это создаёт задержки.
- **Критерий приёмки:**
  - [x] `translate_to_russian()` и `translate_to_english()` переписаны на `httpx.AsyncClient`
  - [x] LRU-кэш заменён на `aiocache` (TTL 24ч, namespace `translator_ru` / `translator_en`)
  - [x] Параллельный перевод title + summary через `asyncio.gather()` в `bot_runner.py`
  - [x] Время перевода 10 статей < 3 секунд (тест `test_parallel_translation_performance`)
  - [x] Обратная совместимость: `translate_to_russian_sync()` / `translate_to_english_sync()` для sync-контекста
  - [x] Graceful shutdown: `close_translator_client()`
- **Файлы:** `translator.py` (переписан), `tests/test_translator.py` (7 тестов)
- **Зависимости:** `aiocache`, `pytest-asyncio`
- **Теги:** `performance`, `async`, `refactoring`

### ✅ P0-003: Graceful degradation RSS-источников (Circuit Breaker) — **ВЫПОЛНЕНО**
- **Дата выполнения:** 2026-06-01
- **Проблема:** При недоступности RSS бот просто логирует warning и продолжает. Нет трекинга здоровья источников.
- **Критерий приёмки:**
  - [x] Класс `SourceHealthTracker` с хранением состояния в SQLite (`source_health`)
  - [x] Состояния: OK → DEGRADED (3 ошибки, 60 мин) → OFFLINE (ещё 3 ошибки, 6 часов)
  - [x] Retry-логика в `RSSParser`: 3 попытки с exponential backoff (2/4/8 сек), таймаут 10 сек
  - [x] В `/health` добавлен раздел со статусом каждого источника (✅/⚠️/🚫)
  - [x] Автоматическое восстановление источника после таймаута
  - [x] Трекинг avg_response_ms для каждого источника
- **Файлы:** `utils/circuit_breaker.py` (230 строк), `parsers/rss_parser.py` (добавлен retry), `tests/test_circuit_breaker.py` (10 тестов)
- **Теги:** `reliability`, `rss`, `monitoring`
